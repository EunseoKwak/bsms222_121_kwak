10.5.2 Which base?

we used base 2 in the above case, and we can also use natural log and base 10. We generally don’t recommend natural log.

we prefer to have smaller integers in the scale - range that includes many integers as possible. Another consequence of limtied range is that choosing the binwidth is more challenging. with log base 2, binwidth of 1 will translate to bin with range x to 2x

filter(gapminder, year == past_year) %>%
  summarize(min = min(population), max = max(population))

for this one, log base 10 is preferable

gapminder %>% 
  filter(year == past_year) %>%
  ggplot(aes(log10(population))) +
  geom_histogram(binwidth = 0.5, color = "black")

10.5.3 transform the values or the scale?

There are two ways we can use log transformation 1) log the values before plotting 2) log scales in the axes.

If we log the data, we can easily interpret intermediate values in the scale.

the advantage of using logged scales is that the original values are displayed in the plot, which are easier to interpret.

If we want to scale the axis with logs, we can use the ’scale_x_continuous_function. Instead of logging the values first, we apply this layer:

gapminder %>% 
  filter(year == past_year & !is.na(gdp)) %>%
  ggplot(aes(dollars_per_day)) + 
  geom_histogram(binwidth = 1, color = "black") +
  scale_x_continuous(trans = "log2")

Not that the log base 10 has own function ‘scale_x_log10()’, but log2 does not so we have to define our own

We can do other transformations through the argument ‘trans’ The square root(sqrt) transformations is useful when considering counts, and logistic transformtion(logit) is useful when plotting proportions between 0 and 1. the ‘reverse’ transformation is useful when we want samller values to be on the right or on top. +) so do we decide if we wanna put smaller values on right or top? or just goes to the upper right part?
